# Blueprint Anatomy (Recommended)

This repository uses a recommended folder structure for blueprints.

It is **not** a rigid standard. Blueprints can evolve based on artifact type, team workflow, and delivery context.

## Recommended files

### `README.md`
Explains what the blueprint creates, who it is for, and how to use it.

### `intent.md`
Defines the job the artifact is designed to accomplish.

### `content.md`
Describes source content, required inputs, optional inputs, and AI content-generation guidance.

### `style.md`
Defines visual, brand, voice, and formatting guidance.

### `output.md`
Defines the target artifact shape, sections, constraints, and acceptance criteria.

### `examples/`
Stores reference outputs, diagrams, screenshots, PDFs, or other files that help AI understand the desired result.
