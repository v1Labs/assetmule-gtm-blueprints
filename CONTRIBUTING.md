# Contributing to AssetMule GTM Blueprints

Thanks for contributing.

## How to propose a new blueprint

1. Open an issue describing the artifact and the GTM workflow it supports.
2. Add a new folder under `blueprints/` with clear, minimal markdown files.
3. Include practical usage guidance and acceptance criteria.
4. Add or update examples when they improve clarity.

## How to improve an existing blueprint

- Submit focused pull requests with clear before/after improvements.
- Keep edits practical and easy to apply in real workflows.
- Tighten unclear instructions, remove noise, and improve scannability.

## Contribution principles

- Keep blueprints practical and easy to use.
- Prefer minimal, clear markdown over complex schemas.
- Avoid overfitting to one company unless the blueprint is explicitly company-specific.
- Do not include private, proprietary, or sensitive client information.
